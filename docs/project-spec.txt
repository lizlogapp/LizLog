蜥日日記(LizLog) 專案計畫書
文件版本：2.0
第一章：策略藍圖
1.1 專案摘要與動機
「蜥日日記」是一款專為鬆獅蜥飼主設計的行動應用程式，旨在結合物聯網（IoT）技術，提供一個集情感紀錄、健康追蹤、協同照護於一體的數位照護平台。
面對多數飼主以手動記錄導致資料零散、醫療過程缺乏依據的痛點，本專案致力於打造一個結合「自動感測」、「飼養日誌」與「健康管理」的一站式工具，協助飼主更科學、細緻地管理寵物的日常與健康。
1.2 核心價值主張
1.從「焦慮」到「自信」：透過系統性的紀錄與智慧提醒，將飼養過程中的不確定	性轉化為可靠數據，賦予飼主自信。
2.從「孤立」到「協作」：透過即時同步的協同照護功能，將家庭成員打造成緊密	的照護團隊，確保照護品質的一致性。
3.從「感覺」到「洞察」：透過數據整合與可視化分析，將飼主的直覺感受提升為	有據可循的科學洞察，輔助更精準的決策。

1.3 目標客群
1.主要客群 - 新手及愛好者：首次飼養，充滿熱情並希望詳盡紀錄寵物成長的新	手；以及飼養數量較少，追求精細化管理的資深飼主，特別是需要家庭成員共同	照護的用戶。
2.次要客群 - 獸醫師或寵物健康專家：可利用飼主分享的數據報告，作為寵物問	診時的輔助資訊，提升診斷效率與準確性。
第二章：產品規格
2.1 核心功能列表
1.多寵物管理系統：支援建立與管理多個寵物檔案。
2.整合式每日照護紀錄：飲食、排泄、日常活動（泡澡、日照、活動力）、健康觀	察（蛻皮、體重、身長）、備註功能：提供自由文字欄位，供使用者紀錄任何額	外描述。
3.日記功能：支援文字、照片、檔案上傳，並能自動生成「當日數據快照」，連結	情感與數據。
4.醫療紀錄模組：結構化記錄就診、用藥史，並可與提醒功能連動。
5.自訂提醒功能：支援單次與重複性提醒，並與紀錄功能形成閉環體驗。
6.協同照護：支援多位家庭成員共享、同步所有寵物檔案與紀錄。
2.2 數據可視化與分析
1.環境趨勢圖：將溫/濕度、日照時數等數據轉換為視覺化圖表。
2.成長曲線圖：繪製體重與身長隨時間變化的成長曲線。
3.關聯對比圖：允許用戶將兩種數據（如：環境vs.行為）疊加在同一圖表上進行對比分析。
4.行為頻率日曆：以日曆熱點圖形式，呈現特定照護事件的頻率。
2.3 用戶介面 (UI) 設計目標
1.設計目標：數據清晰，體驗溫暖。
2.風格方向：介面骨架專業清晰，資訊呈現乾淨直觀；介面細節溫暖，透過客製化	圖示、柔和色彩營造溫馨感。
3.色彩基調：以暖黃、橘色系為主，搭配米白或淺灰色背景。
4.核心體驗：可靠的溫暖感。
第三章：技術與執行
3.1 軟硬體技術架構
採用業界主流的「前後端分離」與「無伺服器 (Serverless)」架構，並導入AI輔助開發流程(AI-Assisted Development)，在確保系統穩定性與擴充性的同時，大幅降低開發與維運成本。
1.行動應用端：
a)開發框架：React Native (Expo)，全球市佔率最高的跨平台框架。透過Expo生態系，無需複雜的原生環境設定，即可使用同一套代碼庫 (Codebase) 同時發布 iOS 與 Android 雙平台應用。
b)程式語言：JavaScript (ES6+)，語法簡潔直觀，擁有龐大的開源資源庫。配合AI工具可快速生成UI組件與邏輯。
c)開發環境(IDE)：VS Code 搭配 Cursor (AI)，利用Cursor的AI引擎進行「結對程式設計 (Pair Programming)」，加速程式碼撰寫、除錯與重構效率。

2.雲端後端與資料庫
a)核心平台： Google Firebase (Serverless 架構)：採用Firebase免費方案，初期零成本，且無需自行架設或維護實體伺服器。
b)身份驗證 (Authentication)：整合 Google Sign-In，提供安全且快速的一鍵登入體驗，並自動處理使用者帳戶管理。
c)資料庫：Firestore (NoSQL)，用於儲存結構化資料（如：寵物檔案、日記文字、健康紀錄），具備靈活的文件結構。
d)Realtime Database：專用於IoT數據同步，能以毫秒級速度將飼養箱的溫濕度數據推送到使用者手機。
e)媒體儲存：用於儲存使用者上傳的寵物照片、蛻皮紀錄與醫療單據影像。

3.物聯網硬體端 (IoT Architecture)
a)微控制器 (MCU)： ESP32-WROOM
b)規格：雙核心處理器，內建Wi-Fi與藍牙，具備低功耗與高性價比特性。
c)開發語言：C++ (Arduino Framework)。利用開源社群豐富的驅動庫，快速實現硬體控制。
d)感測模組：SHT31 / SHT40
e)通訊協定：硬體透過 Wi-Fi 直接呼叫 Firebase REST API，將環境數據寫入雲端，實現「設備直連雲端 (Direct-to-Cloud)」的高效傳輸。
3.2 App 導航結構
採用五個底部頁籤的「平衡功能模型」：
1.首頁：資訊匯總中心，顯示即時數據、今日待辦提醒、最新動態。
2.紀錄：統一的歷史回顧時間軸，是新增「數據紀錄」與「日記」的核心入口。
3.分析：數據可視化中心，提供各類分析圖表的入口。
4.寵物：管理所有寵物檔案，並作為「提醒管理」、「醫護紀錄」、「協同照	護」等功能的入口。
5.設定：標準設定列表，管理帳號、通知與IoT設備。
3.3 核心互動設計原則
1.清晰先於美觀：確保數據和圖表的易讀性。
2.引導而非強迫：溫和引導新用戶，但不妨礙熟悉的使用者。
3.給予即時回饋：操作後給予立即、清晰的正面回饋。
4.操作可預期且可撤銷：設計應符合使用者預期，並為破壞性操作提供防呆與復原		機制。

第四章：未來發展 (The "Where To")
4.1 專案成功指標
1.用戶活躍與留存：監測每日/每週活躍用戶數、用戶留存率。
2.核心功能採用度：追蹤每位用戶平均建立的寵物檔案數、協同照護啟用率、IoT	設備連結率。
3.核心價值實現度：分析「關聯對比圖」等數據分析功能的使用頻率。
4.2 未來展望 (Roadmap)
1.擴大物種支援：將支援範圍擴展至其他爬蟲（豹紋守宮、蛇、龜）及其他類型寵	物。
2.智慧診斷建議：引入機器學習模型，主動發現數據異常並提供初步建議。
3.社群與專家連結：建立飼主社群，並引入認證專家提供線上諮詢服務。
4.硬體生態系整合：從感測擴展至遠端控制加熱、照明、噴淋等設備，成為智慧飼	養環境的控制中樞。
